CI/CD Documentation

This documentation provides an overview of our Continuous Integration and Continuous Deployment (CI/CD) practices.

Overview

Our CI/CD pipeline automates the process of software integration and deployment, ensuring that code changes are tested and deployed efficiently.

Key Features

  • Automatic deployment from GitHub to NFS server.
  • Trigger builds on every code change.
  • Seamless integration with Jenkins for continuous delivery.

Getting Started

To get started with our CI/CD setup, follow these steps:

  • Clone the repository from GitHub.
  • Make changes to the codebase.
  • Push the changes to the main branch.
